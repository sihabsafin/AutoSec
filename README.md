# AutoSEC

**A production-grade multi-agent cybersecurity threat detection and response system built with CrewAI.**

Upload a log file, paste a vulnerability scan, or describe an incident. Get a complete security intelligence package — threat mapping, CVE analysis, incident classification, response playbook, and compliance audit — generated by 13 specialized AI agents across 5 sequential reasoning phases.

> **This is a defensive security analysis tool, not an active scanning or exploitation platform.**
> All findings require verification by a qualified security professional before any remediation action is taken.

## What it does

AutoSEC takes any security data — server logs, Nmap scans, CloudTrail exports, or incident descriptions — and runs it through five sequential AI crews. Each crew has three specialized agents. By the time the pipeline completes, you have:

- MITRE ATT&CK tactical coverage with technique IDs and confidence scoring
- Indicator of Compromise (IOC) table with IP reputation, tool signatures, malicious paths
- CVE mapping against NIST NVD with CVSS scores and exploitability assessment
- Complete attack chain reconstruction from initial access to data exfiltration
- Exposure risk score (0–100) with actionable quick wins
- NIST SP 800-61 incident response playbook tailored to your incident
- Incident priority classification (P1–P4) with regulatory notification requirements
- Executive summary in business risk language (non-technical)
- Full technical report with exact remediation commands and verification steps
- Compliance gap analysis across OWASP, NIST CSF, PCI-DSS, HIPAA, GDPR, ISO 27001
- Downloadable PDF, JSON, and text reports

## The 5-Phase Pipeline

### Phase 1 — Target Intake

| Agent | Responsibility |
|---|---|
| TargetParserAgent | Extracts IPs, domains, ports, file paths, user agents, timestamps from any input format (Apache logs, Nmap, CloudTrail, raw text) |
| ContextBuilderAgent | Determines target criticality, attack surface, data sensitivity classification, compliance scope, likely threat actor profile |
| SecurityContextAgent | Identifies vulnerable versions, misconfigurations, compliance violations, and initial risk posture |

### Phase 2 — Threat Detection

| Agent | Responsibility |
|---|---|
| ThreatClassifierAgent | Maps indicators to MITRE ATT&CK tactics and techniques with confidence scoring and severity ratings |
| IOCAnalyzerAgent | Identifies suspicious IPs, attack tool signatures, malicious paths, authentication anomalies, data exfiltration patterns |
| AnomalyDetectorAgent | Detects behavioral red flags: brute force, privilege escalation, lateral movement, webshells, crypto mining |

### Phase 3 — Vulnerability Assessment

| Agent | Responsibility |
|---|---|
| VulnerabilityMapperAgent | Maps services/versions to CVEs with CVSS scores, exploitability ratings, and available patches |
| AttackVectorAnalyzerAgent | Constructs complete attack chains with likelihood scores, blast radius, and business impact assessment |
| ExposureAssessorAgent | Calculates exposure risk score (0–100) across 5 dimensions: access, criticality, exploitability, detectability, remediability |

### Phase 4 — Incident Response

| Agent | Responsibility |
|---|---|
| IncidentClassifierAgent | NIST/SANS incident classification with regulatory notification assessment (GDPR, HIPAA, NERC-CIP, SOX) |
| ResponsePlaybookAgent | Full 6-phase response playbook (preparation, detection, containment, eradication, recovery, post-incident) with owner and timeline |
| RemediationAdvisorAgent | Exact remediation commands with pre/post conditions, verification steps, and documented rollback procedures |

### Phase 5 — Security Report

| Agent | Responsibility |
|---|---|
| ExecutiveReportAgent | Non-technical business risk summary with financial impact, regulatory risk, and board-level recommendations |
| TechnicalReportAgent | Full technical findings with CVE references, ATT&CK mappings, command history, and forensic observations |
| ComplianceMapperAgent | Gap analysis and control mapping across OWASP Top 10, NIST CSF, PCI-DSS, HIPAA, GDPR, ISO 27001 |

## Per-Phase Model Selection

Each phase uses the model you select at runtime. There is no default — you choose before running. This enables flexibility to route different phases to different providers based on performance and rate limits.

Recommended configuration for best results with zero cost:

```
Phase 1 — Target Intake              →  Groq LLaMA 3.3 70B       (fast extraction, high TPM)
Phase 2 — Threat Detection           →  Gemini 2.5 Flash          (strong pattern recognition)
Phase 3 — Vulnerability Assessment   →  Groq Qwen3 32B            (technical depth)
Phase 4 — Incident Response          →  Gemini 2.5 Flash          (long-form reasoning)
Phase 5 — Security Report            →  Groq LLaMA 3.3 70B        (structured output)
```

If you hit a rate limit on any phase, switch to **Gemini Auto** or keep a fallback ready.

## Key Implementation Details

**Lazy loading** — Every agent is wrapped in a getter function. No agent is instantiated at module level. This prevents the `OPENAI_API_KEY is required` crash that occurs when Streamlit Cloud loads `st.secrets` before module-level code runs.

```python
# Wrong — crashes on Streamlit Cloud
llm = get_llm()
parser_agent = Agent(role="...", llm=llm)

# Correct
def get_target_parser_agent(phase_key: str = "phase1") -> Agent:
    return Agent(role="...", llm=_llm(phase_key), ...)
```

**Task parameter pattern** — Tasks receive their agent as a parameter rather than importing module-level instances.

```python
def task_parse_target(agent, target_input: str) -> Task:
    return Task(description=f"...", agent=agent)
```

**Safe JSON parsing** — Every agent output goes through `_safe_json()` before use. Agent outputs are not always valid JSON — this handles markdown fences, truncated output, and type mismatches.

```python
def _safe_json(raw) -> dict:
    try:
        text = str(raw).strip()
        if "```json" in text:
            text = text.split("```json")[1].split("```")[0].strip()
        elif "```" in text:
            text = text.split("```")[1].split("```")[0].strip()
        return json.loads(text)
    except Exception:
        return {"raw_output": str(raw)}
```

**Multi-format input parsing** — The `input_parser.py` module detects input type (Apache log, Nmap scan, CloudTrail, free text) and applies format-specific regex before passing to agents. This reduces token usage and improves accuracy.

**Temperature 0.2** — All agents run at temperature 0.2 to minimize hallucination. Security analysis requires deterministic, fact-grounded output. A CVE ID is either correct or it isn't.

## Rate Limits — Handling and Strategy

**Groq TPM exceeded (14,400 req/day)** — Switch the current phase to Gemini. Groq's free tier resets every 24 hours. One full 5-phase analysis uses ~15–20 API calls, meaning you can run 700–900 complete analyses per day.

**Gemini quota exceeded (1,500 req/day)** — Switch to Groq for the next phase. Gemini's limit resets daily at midnight UTC.

**NIST NVD API rate limited (5 req/30s)** — The code handles this automatically with `time.sleep(6)` between CVE lookups. Very large Nmap scans (100+ services) may take several minutes for the CVE phase.

**General strategy:**
1. Run Phase 1 and 2 with Groq (fast, high TPM)
2. Run Phase 3 with Gemini (strong vulnerability reasoning)
3. Run Phase 4 and 5 with Groq (structured output)
4. If any phase hits limits, the UI shows which model to switch to next

## Architecture Decisions & Rationale

**Sequential phases, not parallel** — Each phase feeds structured output into the next. Threat classification informs vulnerability prioritization. Vulnerability data shapes the incident response playbook. Parallel execution breaks this dependency chain. Sequential also makes debugging straightforward — you can inspect exactly what each agent produced.

**Lazy loading mandatory** — CrewAI agents instantiate the LLM client on creation. On Streamlit Cloud, `st.secrets` is not available at module import time — only at runtime. Any agent created at module level will crash with a missing API key error before the app even starts. All 13 agents use getter functions that run only when called.

**components.html() over st.markdown()** — Streamlit's `unsafe_allow_html=True` works for simple cases but fails silently for complex HTML inside column loops, animated elements, and custom styling. `components.html()` renders in an isolated iframe with full CSS support. Every card and terminal in AutoSEC uses it.

**Temperature 0.2** — Security analysis requires deterministic, precise output — not creative variation. A CVE ID is either correct or it's not. An attack chain either follows the evidence or it fabricates. Temperature 0.2 keeps agents grounded in the data.

**Input parsing before LLM calls** — The regex-based parser in `input_parser.py` runs locally in milliseconds and extracts structured indicators before any LLM call. This gives agents pre-processed, structured data rather than raw text, improving accuracy and reducing token usage.

## Known Limitations

**No active scanning.** AutoSEC analyzes data you provide. It does not connect to targets, run Nmap, or perform any active reconnaissance. If you want scan data, run the scan yourself and paste the output.

**AI findings require expert review.** The agents can misidentify benign traffic as malicious, miss context that a human analyst would catch, or produce CVE mappings that don't apply to your exact environment. Treat output as a high-confidence starting point for investigation, not a definitive verdict.

**NVD API rate limits.** Free NIST NVD allows 5 requests per 30 seconds without an API key. Large Nmap scans (100+ services) will take several minutes for the CVE lookup phase due to built-in rate limit handling.

**Groq free tier limits.** 14,400 requests per day. One full 5-phase analysis uses ~15–20 calls. You can run 700–900 complete analyses per day on the free tier.

**Context window.** Large log files are truncated at 8,000–10,000 characters for the parser and 5,000–6,000 for downstream agents. Pre-filter logs to the suspicious time window before pasting.

## Troubleshooting

**App crashes immediately on Streamlit Cloud**

Check that Python version is set to 3.11 in App Settings → General. The default is 3.9 which has dependency compatibility issues.

**"OPENAI_API_KEY is required" error**

An agent or dependency is being instantiated at module level before `st.secrets` is available. All agents in `agents.py` must use getter functions. Verify no agent is created outside a function.

**GROQ_API_KEY not found despite being in secrets**

Confirm the key is in Streamlit Cloud Secrets (Settings → Secrets), not just in your local `.streamlit/secrets.toml`. Local files are not deployed.

**NVD API returning empty results**

NIST NVD rate limits anonymous requests. If running multiple searches quickly, results may be delayed. The pipeline automatically adds `time.sleep(6)` between CVE lookups. Manual searches from the UI do not have retry logic — refresh after 30 seconds.

**PDF generation fails with font error**

ReportLab uses standard built-in fonts (Helvetica, Courier) and does not require system font installation. If you see a font error, verify `reportlab==4.2.5` installed correctly.

**Supabase connection refused**

Verify `SUPABASE_URL` includes `https://` and ends without a trailing slash. Verify `SUPABASE_KEY` is the `anon public` key, not the `service_role` key. AutoSEC gracefully skips database operations if the connection fails — the app continues working with session state only.
